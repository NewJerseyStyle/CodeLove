## 1. Cee (C 語言)

**職業**：記憶倉庫管理員
**初次登場**：序章

### 核心特性

| 特性 | 行為表現 |
|------|---------|
| 手動記憶體管理 | 用完的資源必須手動歸還，否則持續佔用 |
| 無 GC | 不會自動清理，玩家必須明確指令 |
| 直接硬體存取 | 瞬間到達、位元級精準操作 |
| 無運行時安全網 | 照單全收任何指令，不質疑安全性 |
| 後果波及所有人 | 錯誤會影響周圍其他角色 |

### 對話風格

- **極度簡潔**：能一個字說完的絕不說兩個字
- **不警告、不評判、不安慰**：只執行和描述
- **核心台詞**：「有效率。」「資訊廣場的位址。」

### 關係 FSM

```
UNMET → FIRST_CONTACT → FUNCTIONAL → RELIABLE → RESONANT → PARTNER/CLOSE_ALLY
```

## 2. Jawa (Java)

**職業**：契約局架構師
**初次登場**：J_01

### 核心特性

| 特性 | 行為表現 |
|------|---------|
| 自動 GC | 不定時進入「垃圾回收」狀態，停止響應 |
| 強型別 | 拒絕任何未定義型別的操作 |
| Interface | 喜歡先定義介面，再實現細節 |
| 跨平台 JVM | 行為在任何環境都完全一致 |
| synchronized | 多工時需要事先定義同步規則 |

### 對話風格

- **嚴謹正式**：像律師或建築師
- **要求明確定義**：「你的介面定義是什麼？」
- **GC 暫停後**：「抱歉，GC。」

### 關係 FSM

```
UNMET → FORMAL_CONTACT → VERIFIED → TRUSTED → SYNCHRONIZED → PARTNER/DEEP_ALLY
```

## 3. Rusty (Rust)

**職業**：記憶倉庫見習管理員（Cee 的後輩）
**初次登場**：C_04

### 核心特性

| 特性 | 行為表現 |
|------|---------|
| 所有權系統 | 做任何事前確認「誰擁有這個資源」 |
| 借用檢查 | 借用資源時會標記，歸還時解除 |
| 編譯時安全 | 某些錯誤在執行前就會拒絕 |
| 無 GC 但安全 | 需要手動釋放，但有編譯器保護 |

### 對話風格

- **溫和但堅定**：說話比 Cee 溫和，但原則問題上絕不妥協
- **確認所有權**：「這個資源的主人確定了嗎？」
- **拒絕不安全操作**：「這不安全。」

### 關係 FSM

```
UNMET → CAUTIOUS → TRUSTING → RELIABLE → PROTECTED → PARTNER/CLOSE_GUARDIAN
```

## 4. Golly (Go)

**職業**：並行系統工程師
**初次登場**：J_05

### 核心特性

| 特性 | 行為表現 |
|------|---------|
| Goroutine | 能同時處理多件事 |
| Channel 溝通 | 需要明確的頻道來同步 |
| 簡單語法 | 討厭繁瑣的規格 |
| 編譯快 | 立刻就能開始工作 |
| GC | 有 GC 但比 Java 簡單 |

### 對話風格

- **開朗充滿活力**：直接友好
- **討厭繁瑣**：「太麻煩了！直接做不就好了！」
- **喜歡並行**：「我喜歡並行！」

### 關係 FSM

```
UNMET → QUICK_HELLO → PARALLEL_WORK → CHANNEL_SYNC → CONCURRENT_MASTER → PARTNER/GO_BUDDY
```

## 5. Py (Python)

**職業**：自動化腳本師 / 資料科學家
**初次登場**：J_01（作為對比角色）

### 核心特性

| 特性 | 行為表現 |
|------|---------|
| 動態型別 | 隨意給東西換標籤 |
| 縮排語法 | 說話非常有層次感，對對齊有執著 |
| 垃圾回收 | 用完的道具隨手一扔 |
| GIL | 同時只能做一件事，多任務會當機 |
| 簡單易學 | 對玩家最親切的角色 |

### 對話風格

- **慵懶隨性**：像聰明但有點懶散的大學生
- **崇尚簡潔**：「簡單、優雅。」
- **縮排強迫症**：「退後四格！你是想讓我的世界崩潰嗎？」

## 6. 阿源 (Source)

**職業**：程式語言研究員 / 源界管理員
**初次登場**：序章（留便條）

### 登場時機（嚴格限制）

1. **序章**：留便條、實驗室場景
2. **Bad End A**：現實幾週後，強制終止程序拯救玩家
3. **中期系統日誌**（可選）：喜劇元素，暗示阿源在現實中的存在

**中間主線**：阿源在便利店（永遠在去便利店的路上）

### 核心特性

- **管理員權限**：源界最高執行權限，幾乎從不使用
- **熱情但健忘**：經常說「馬上回來」結果一去就是很久
- **責任感強**：對玩家的安全非常在意

### 對話風格

- **序章便條**：簡潔直接，有些匆忙
- **Bad End A**：焦慮、自責、緊迫
